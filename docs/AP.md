4. 개발 및 데이터 구축 가이드 (Action Plan)
   Phase 1: 데이터셋 확보 (가장 중요)
   범위가 넓어진 만큼, MVP에서는 각 영역별 대표 제품군만 우선 DB에 등록합니다. (총 50~60개 목표)
   식사(Eating): 만능 커프, 휜 수저, 무거운 수저, 미끄럼 방지 매트.
   목욕/용변(Hygiene): 목욕 의자, 이동 변기, 안전 손잡이.
   이동(Mobility): 휠체어(활동형/일반), 실버카, 지팡이.
   정보/소통(IT/Comm): 트랙볼 마우스, 키가드, 확대 독서기(휴대용).
   착탈의(Dressing): 단추 끼우개, 양말 신기 보조구.
   Phase 2: UI 구현 (Dynamic Form)
   바이브코딩에게 다음과 같이 지시하세요.
   "설문 페이지(Step 2)를 만들 때, Step 1에서 '식사'를 선택한 사람에게는 '손 사용 기능'에 대한 질문만 나오고, '시각'을 선택한 사람에게는 '보이는 정도'에 대한 질문이 나오도록 **조건부 렌더링(Conditional Rendering)**을 적용해줘."
   Phase 3: AI 매칭 로직
   "Gemini가 반환한 JSON의 target_domain과 recommended_category를 이용해서 Supabase products 테이블을 필터링하는 로직을 짜줘. 만약 사용자가 '식사'와 '이동' 두 가지 어려움을 다 입력했다면, 두 영역의 제품을 각각 1개 이상씩 추천해주는 로직이 필요해."
