2. PRD (Product Requirements Document) - 기능 명세
2.1 디딤 체크 (Didim Assessment) - 고도화된 평가 로직
사용자의 피로도를 줄이기 위해 [1단계: 영역 선택] -> [2단계: 상세 질문] 구조로 갑니다.
바이브코딩 프롬프트용 상세 구조:
Step 1: 도움이 필요한 영역 (다중 선택)
🔘 이동 (걷기, 휠체어)
🔘 일상생활 (밥 먹기, 씻기, 옷 입기)
🔘 본다는 것 / 듣는 것 (시각, 청각)
🔘 소통과 컴퓨터 (말하기, PC/스마트폰 사용)
🔘 자세 (앉아있기, 누워있기)
Step 2: 상세 기능 평가 (선택한 영역에 따라 동적으로 질문 노출)
(일상생활 선택 시): "식사하실 때 어떤 점이 힘드신가요? (손 떨림 / 쥐는 힘 부족 / 입까지 가져가기 힘듦)"
(컴퓨터 선택 시): "일반 마우스나 키보드 사용이 어려우신가요? (정확히 누르기 힘듦 / 클릭이 안 됨)"
(시각 선택 시): "어느 정도 보이시나요? (빛만 구분 / 큰 글자만 보임 / 시야가 좁음)"
Step 3: 환경 및 목표
사용 환경: 학교/직장(컴퓨터/이동), 집(식사/목욕), 운전 등.
2.2 결과 리포트 (Result)
다차원 추천: 이동 보조기기뿐만 아니라, 식사 도구 등 **보조 도구(Assistive Device)**를 함께 패키지로 제안 가능.
예: "뇌졸중으로 편마비가 있으신 아버님께는 **한 손 브레이크 휠체어(이동)**와 **미끄럼 방지 접시(식사)**를 함께 추천합니다."
